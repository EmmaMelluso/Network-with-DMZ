REGOLE DI SICUREZZA DA AGGIUNGERE.

1. Le mail devono essere inoltrate usando SMTP in entrambe le direzioni attraverso i firewalls, ma in ogni caso devono passare per il DMZ mail gateway (fornisce sanitization e content filtering). Le mail provenienti dall'esterno sono destinate al DMZ mail server.

2. Gli host interni devono poter recuperare le mail a loro destinate dal DMZ mail gateway, utilizzando sia POP3 che POP3S (???) e autenticarsi.

3. Gli utenti esterni devono poter recuperare le mail a loro destinate dal DMZ mail gateway, ma solo se utilizzano POP3 previa autenticazione.

4. Le web requests (sia insicure che sicure) sono permesse per ogni host interno (devono comunque passare attraverso il firewall) ma devono passare per il DMZ web proxy, il quale fornisce content filtering (questa cosa la si fa solo per le richieste non sicure). Gli host interni devono autenticarsi con il proxy.

5. Le web requests (sia insicure che sicure) sono permesse per ogni host esterno verso il DMZ web server.

6. Le richieste DNS lookup fatte dagli utenti interni sono consentite e devono passare per il DMZ DNS server, il quale inoltra le query alla rete esterna (Internet).

7. Le richieste DNS provenienti dall'esterno sono soddisfatte dal DMZ DNS server.

9. Le management request gestite tramite il protocollo SNMP sono permesse agli host interni che si occupano del management della rete verso i firewalls. Anche i firewall sono configurati in modo da mandare notifiche di particolari eventi ai management hosts.
